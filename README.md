# 📊 Wordstat Analyzer

Веб-сервис для профессиональной работы с поисковым спросом. Автоматизирует взаимодействие с API Яндекс.Вордстата и предоставляет инструменты аналитики, которых нет в стандартном интерфейсе.

## 🚀 В чем преимущество?

* **Массовая обработка:** Анализ множества запросов одновременно. Вместо ручного ввода каждой фразы, сервис обрабатывает их пачкой, экономя часы рутинной работы.
* **Личный кабинет и История:** Полноценная многопользовательская система. Все ваши поиски сохраняются в персональной истории: к ним можно вернуться спустя месяц, сравнить показатели или выгрузить отчет повторно, не тратя лимиты API.
* **Умный экспорт в Excel:** Формирование единого файла на всю группу запросов. Каждый отчет аккуратно разносится по вложенным листам -  не нужно скачивать и склеивать отдельные файлы вручную.
* **Визуализация данных:** Наглядные графики и диаграммы для анализа трендов и долей рынка, которые нет в стандартном API Яндекс.Вордстата.

   
## ✨ Ключевые возможности

* **Batch-обработка:** Группировка нескольких фраз в один аналитический отчет.
* **Географический анализ:** Распределение запросов по городам и странам.
* **Сегментация по устройствам:** Раздельный анализ трафика: десктопы, мобильные устройства и планшеты.
* **Анализ динамики:** Отслеживание трендов спроса по неделям и месяцам для выявления сезонности.
* **Хранение:** Собственная база данных для хранения истории запросов.
* **Адаптивный UI:** Лаконичный интерфейс, который подстраивается под тип задачи.

## 🛠 Технологический стек

* **Backend:** Python, **FastAPI**.
* **Database:** **PostgreSQL** + **SQLAlchemy** (ORM) + **Alembic**.
* **Frontend:** Vanilla JS, HTML5, CSS3 (Modern Flexbox/Grid).
* **DevOps:** **Docker** & **Docker Compose**.
* **Validation:** **Pydantic**.

## 📂 Архитектура проекта

* `app/api/` - Роутеры и эндпоинты.
* `app/services/` - Бизнес-логика и интеграция с внешним API Яндекса.
* `app/crud/` - Оптимизированные асинхронные запросы к БД.
* `app/schemas/` - Строгая валидация входящих и исходящих данных через Pydantic.
* `app/models.py` - Описание структуры таблиц (SQLAlchemy).

## 🚀 Запуск

1.  Клонируйте репозиторий.
2.  Создайте файл `.env` на основе `.env.example` и добавьте свой API-ключ.
3.  Перейдите в ветку `dev`.
4.  Запустите проект командой:
    ```bash
    docker-compose up --build
    ```
5.  Сервис будет доступен по адресу: `http://localhost:8000`
