# 📊 Wordstat Analyzer

Веб-сервис для работы с поисковым спросом, автоматизирует взаимодействие с API Яндекс.Вордстата, делает его более удобным.

## Премущества над обычным Wordstat API?

1.  Вместо написания скриптов для обработки JSON, вы получаете готовый интерфейс с валидацией данных.
2.  Результаты запросов кэшируются в БД. Повторный просмотр истории не тратит платные баллы API.
3.  Не нужен Excel, чтобы посчитать долю региона или Affinity Index - сервис делает это на лету.
4.  API возвращает только коды регионов, сервис сам находит и подставляет названия для улучшения читаемости.
   
## ✨ Ключевые возможности

* **Географический анализ:** Распределение запросов по городам и странам с автоматическим расчетом **Affinity Index** (индекс популярности).
* **Сегментация по устройствам:** Раздельный анализ трафика: десктопы, мобильные устройства и планшеты.
* **Анализ динамики:** Отслеживание трендов спроса по неделям и месяцам для выявления сезонности.
* **Хранение:** Собственная база данных для хранения истории запросов.
* **Адаптивный UI:** Лаконичный интерфейс, который подстраивается под тип задачи.

## 🛠 Технологический стек

* **Backend:** Python, **FastAPI**.
* **Database:** **PostgreSQL** + **SQLAlchemy** (ORM) + **Alembic**.
* **Frontend:** Vanilla JS, HTML5, CSS3 (Modern Flexbox/Grid).
* **DevOps:** **Docker** & **Docker Compose**.
* **Validation:** **Pydantic**.

## 📂 Архитектура проекта

* `app/api/` — Роутеры и эндпоинты.
* `app/services/` — Бизнес-логика и интеграция с внешним API Яндекса.
* `app/crud/` — Оптимизированные асинхронные запросы к БД.
* `app/schemas/` — Строгая валидация входящих и исходящих данных через Pydantic.
* `app/models.py` — Описание структуры таблиц (SQLAlchemy).

## 🚀 Запуск

1.  Клонируйте репозиторий.
2.  Создайте файл `.env` на основе `.env.example` и добавьте свой API-ключ.
3.  Запустите проект командой:
    ```bash
    docker-compose up --build
    ```
4.  Сервис будет доступен по адресу: `http://localhost:8000`
